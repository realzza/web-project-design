Functions, Methods, and Events

Functions are blocks of code (subprograms) designed to perform particular tasks, often ones you want to perform over and over again. Functions are executed when something invokes or calls them.
Parameters are values that can be passed into the function and used within it:

function myFunction(param1,param2){
return param1 + param2;
}

If you invoke the above with myFunction(5,6);, it will return the value 11.

Methods

Methods are functions defined within objects that can do things with that object.

const person = {
firstName: "Matilda",
lastName: "Smith",
age: 15,
occupation: "student",
hometown: "Springfield, OH",
introduction: function(){
return this.firstName + " " + this.lastName + " is a " + this.occupation + " from " + this.hometown + "."
},
}

Methods are often written with the "this" keyword, which refers to the "owner" of the function, the object.

To invoke the above function on person and log it to the console you could say:console.log(person.introduction());

Events

The browser and window are contexts of a lot of activity: clicking, scrolling, hovering, etc., and JavaScript events harness that activity to trigger code.

Clicking on this div, for example, will trigger a click event.

JavaScript events can be handled either by 1. using an event handler, or by adding an event listener. See the .js file for examples. The key difference is that event handlers can only be called once on the same object for the same event, whereas event listeners can be used together on the same object for the same event to trigger multiple actions.
